Introduction

Welcome to the documentation of Artificial Reason template. In this guide you will find everything needed to install and configure all aspects of the template.

Package Content

Package content is:

  • artificial-reason-grunt folder
    • css folder - css template files
    • less folder - less template files
    • font folder - font files used in the template.
    • html folder - all html files (Raw) are available.
    • Folders: img, js, less - Different image files, javascript and find less uncompiled.
    • bower.js - Bower configuration file contains the template dependencies.
    • Gruntfile.js - Grunt configuration file.
    • package.json - Configuration File node.js.
  • artificial-reason-gulp folder
    • src folder
      • css folder - css template files
      • less folder - less template files
      • font folder - font files used in the template.
      • html folder - all html files (Raw) are available.
      • Folders: img, js, less - Different image files, javascript and find less uncompiled.
    • bower.js - Bower configuration file contains the template dependencies.
    • gulpfile.js - Gulp configuration file.
    • package.json - Configuration File node.js.
  • README.txt - Readme highly recommended reading.
  • changelog.txt - File with the change history of all updates.
  • Credits.txt - Credits has third party content.
  • dist folder - Contains all final theme files ready to use, unless you want to configure anything to use the files in this folder.
  • Documentation folder - all documentation template is located.

Installation

You have two options: Use the compiled template or generate it yourself with grunt.

Use the compiled template

  1. Create a folder where you want to have your installation.
  2. Copy the contents of the following folder: dist/assets in your installation folder
  3. Copy to your install folder all the html files in a folder found in dist/ According to the header you choose.
  4. Ready to use!

Build the template with Bower and Grunt

To use this option you must have installed node.js, Bower and Grunt. (Bower requires using git to download the packages).

Once installed the necessary tools, with a console command sure that you are in the template folder and run the following commands:

npm install
bower install
grunt dist

This will generate the dist folder with all the processed content, then follow the instructions on the first option.

If you want to have an alternative method where all the file paths are relative and not have to copy anything from the asset folder to run the template so you can test all types of header, you should change the last command by following:

grunt dev

Now you can directly upload the contents of the dist directory and everything works properly. You can use this option to modify the Gruntfile.js file and set your own path relative to assets, it should only change the relativeAssetsPath variable.

Build the template with Bower and Gulp

The process is the same as for grunt. Simply install gulp instead of grunt and use the following commands.

npm install
bower install
gulp

Setting a default color scheme

You have two options:

Compiled files

  1. Open a file html
  2. Locate the following line: <link href = "css/style-blue.css" rel = "stylesheet" media = "screen" title = "default">
  3. Change style-blue.css by any color you want. The colors available are in the folder dist/css
  4. Repeat for all html files (We recommend using a text editor with replacement text documents).

Grunt Option (recommended)

  1. Edit html/layout/header/head.html file.
  2. Change the line <link href = "<% =% assets> css / style-blue.css" rel = "stylesheet" media = "screen" title = "default"> replacing style-blue.css by color you want. The colors available are in the folder less/colors
  3. Save the file and now do the grunt compilation process (see installation grunt).

Make a new color scheme

To create a new color scheme is necessary to use compilation with Grunt.

  1. Go to the less/colors folder.
  2. Copy one of the files and rename it with the name you want, example: newColor.less
  3. Edit the new file and set its new color scheme by changing the value of the variables. Save the file.
  4. Edit Gruntfile.js file located in the root directory. Add the new color with the name of the file created in the previous step: 
    colors: {
  aqua: {}, blue: {}, blue2: {}, blue3: {}, blue4: {}, blue5: {}, fuchsia: {}, gray: {},
  green: {}, green2: {}, green3: {}, green4: {}, green5: {}, orange: {}, orange2: {},
  orange3: {}, pink: {}, red: {}, red2: {}, red3: {}, violet: {}, violet2: {}, violet3: {},
  yellow: {}, yellow2: {}, newColor:{}
},
  5. Save the file. You can now follow the instructions above to set it by default and compile your template with your new color scheme.

Enable/Disable Configurator

To remove the floating panel with color picker follow these steps:

  1. Edit file html/layout/header.html
  2. Delete line: <!-- build:include header/configurator.html --><!-- /build -->
  3. Save the file and follow the orders of usual compilation.
